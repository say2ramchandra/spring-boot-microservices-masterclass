⚙️ Configuration Required

To use this demo with real OAuth2 providers, you need to:

  1. Create OAuth2 app on Google Cloud Console
  2. Create OAuth app on GitHub Developer Settings
  3. Set environment variables: 
    GOOGLE_CLIENT_ID=your-google-client-id
GOOGLE_CLIENT_SECRET=your-google-client-secret
GITHUB_CLIENT_ID=your-github-client-id
GITHUB_CLIENT_SECRET=your-github-client-secret
  4. Set callback URLs:
    • Google: http://localhost:8087/login/oauth2/code/google
    • GitHub: http://localhost:8087/login/oauth2/code/github

📚 OAuth2 Flow Explained

1. User clicks "Login with Google/GitHub"
           ↓
2. Redirect to provider's authorization endpoint
   https://accounts.google.com/o/oauth2/auth?...
           ↓
3. User authenticates and grants permissions
           ↓
4. Provider redirects back with authorization code
   http://localhost:8087/login/oauth2/code/google?code=...
           ↓
5. Spring Security exchanges code for tokens
   (Access Token + Refresh Token)
           ↓
6. Spring Security fetches user info from provider
           ↓
7. CustomOAuth2UserService saves user to database
           ↓
8. User is authenticated and redirected to dashboard
